# 답변 진행방식
1. fork를 한다.
2. 주어진 문장 하위에 [본인 깃허브 닉네임]/README.md를 만든다.
```
defalut
  |-자기_소개를_해주세요
    |-phjppo0918
      |-README.MD
```
3. 생성한 리드미에 답변을 작성한다
4. PR을 날린다

# 질문 추가 진행방식
1. fork를 한다
2. 질문 텍스트를 띄어쓰기를 언더바(_)로 대체하여 폴더명으로 설정한다.
3. [질문텍스트]/README.md 를 만든다
```
network
  |-TCP_와_UDP의_차이를_설명해주세요
    |-README.MD
```
4. (Option)README.md 에 세부적인 질문 사항을 적는다
5. PR을 날린다
